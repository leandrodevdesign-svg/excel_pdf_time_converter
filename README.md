# excel_pdf_time_converter

`excel_pdf_time_converter` is an internal browser-based workspace for reviewing worked hours, converting invoice PDFs into a structured CSV, and comparing Kimai exports against invoice totals.

The project is intentionally lightweight:

- no frontend build step
- static HTML/CSS/JS pages
- client-side parsing for Excel, CSV, and PDF files
- one Vercel serverless route for the Gemini-powered invoice extraction flow

## What The Software Does

This repository currently provides five user-facing pages:

### Home

`index.html`

The home page explains the available workflows and routes teammates to the right page depending on the task.

### Single

`dashboard.html`

Single mode is the one-file version of the dashboard. It is used to inspect one Excel or Kimai-style export and review:

- total hours
- average per day
- busiest day
- billable percentage
- daily distribution
- hours by project
- hours by activity
- detailed records

### Batch

`dashboard.html`

Batch mode is the multi-user or multi-file version of the same dashboard. It uses the same page and logic as Single, but switches to batch behavior through the `#batch` hash and user filters.

Use Batch when:

- one file contains several users
- several files need to be combined
- the team needs to filter visible users before exporting or reviewing data

### Client Overview

`client-overview.html`

Client Overview reads the current Batch snapshot from session storage and renders client-level analytics such as:

- total hours by client
- distribution by client
- workload trend
- activity distribution
- summary management table

### Invoices to Excel

`invoices-to-excel.html`

This page accepts a bulk set of invoice PDFs, extracts their text in the browser, builds an intermediate CSV, sends it to Gemini through the backend route, and returns a final CSV with:

- `Document Name`
- `User Name`
- `Hours Billed`

The user can preview both the source CSV and the final CSV before downloading.

### Kimai vs Invoices

`kimai-vs-invoices.html`

This page compares:

- one Kimai export in `.xlsx` or `.xls`
- one invoice CSV generated by the invoice flow

It normalizes names like `marcelo.fernandez` and `Marcelo Fernandez`, aggregates Kimai hours per user, preserves invoice rows per document, and renders a mirrored comparison table so teammates can reconcile worked hours vs invoiced hours quickly.

## Current Workflow Summary

The software supports these common flows:

1. Upload one timesheet export in Single and inspect the result.
2. Upload several timesheet exports in Batch and filter users.
3. Open Client Overview from Batch to review client-level summaries.
4. Upload invoice PDFs in Invoices to Excel and download the resulting invoice CSV.
5. Upload a Kimai export plus the invoice CSV in Kimai vs Invoices to compare totals side by side.

## Important Implementation Notes

### Single and Batch share one page

`Single` and `Batch` are not separate HTML files. Both use `dashboard.html` and the same script `assets/js/app.js`.
The active mode is controlled by the URL hash:

- `#single`
- `#batch`

This is intentional. It avoids duplicating almost identical dashboard markup and business logic.

### Shared loading and text effects

The project now includes shared page-level effects:

- `assets/js/page-loader.js`
- `assets/css/page-effects.css`

These files provide:

- the page loader shown during initial page load
- shared typewriter styles

Dynamic typewriter behavior is implemented in the page-specific scripts when file uploads or snapshots update the UI.

## Run Locally

### For UI review only

You can serve the static files locally with a simple server:

```bash
python3 -m http.server 8000
```

Then open:

```text
http://localhost:8000
```

This is enough to test:

- Home
- Single
- Batch
- Client Overview
- Kimai vs Invoices

### For end-to-end invoice extraction

The `Invoices to Excel` Gemini flow needs the backend route `api/invoices-to-excel.js`, so a plain `file://` preview is not enough.

For that flow, run the project through Vercel locally or deploy it to Vercel.

## Environment Variables

Required for the invoice AI flow:

```bash
GEMINI_API_KEY=your_key_here
```

Optional:

```bash
GEMINI_MODEL=gemini-flash-latest
```

If `GEMINI_MODEL` is omitted, the backend defaults to `gemini-flash-latest`.

See also:

- `.env.example`

## Input Expectations

### Dashboard pages

The dashboard expects `.xlsx` or `.xls` files whose first sheet contains columns such as:

- `Date`
- `In`
- `Out`
- `Time`
- `Billable`
- `Customer`
- `Project`
- `Activity`
- `Description`
- `Username`

The parser also accepts common English and Spanish variants. See `docs/EXCEL_INPUT.md`.

### Invoices to Excel

The invoice page accepts a bulk set of `.pdf` files.

The browser:

1. extracts machine-readable text with `pdf.js`
2. builds a source CSV with `Document Name` and `Content`
3. sends that CSV to the Gemini backend
4. receives a structured CSV with `Document Name`, `User Name`, and `Hours Billed`

### Kimai vs Invoices

This comparison page expects:

1. one Kimai export in `.xlsx` or `.xls`
2. one invoice CSV with these columns:
   - `Document Name`
   - `User Name`
   - `Hours Billed`

## Main Dependencies

This project loads browser libraries from CDNs instead of using a build step.

Main runtime dependencies:

- `xlsx`: reads Excel workbooks in the browser
- `html2pdf.js`: exports dashboard and overview pages to PDF
- `pdf.js`: extracts invoice text in the browser
- Gemini API: structures invoice text into the final invoice CSV

## Key Files For Developers

### `assets/js/app.js`

Single and Batch dashboard logic:

- file upload handling
- row normalization
- KPI calculation
- chart rendering
- table rendering
- PDF export
- typewriter behavior after uploads

### `assets/js/client-overview.js`

Client Overview logic:

- reads the Batch snapshot from session storage
- computes client-level metrics
- renders ApexCharts charts
- exports the overview as PDF

### `assets/js/invoices-to-excel.js`

Invoice conversion logic:

- bulk PDF selection
- text extraction via `pdf.js`
- source CSV creation
- backend request handling
- final CSV preview and download

### `api/invoices-to-excel.js`

Serverless backend for Gemini:

- receives the source CSV
- sends the prompt to Gemini
- normalizes the AI response
- returns the final CSV plus structured rows
- exposes a small debug GET endpoint for backend verification

### `assets/js/kimai-vs-invoices.js`

Comparison logic:

- parses Kimai workbook totals
- parses invoice CSV rows
- normalizes names across both sources
- builds mirrored groups and deltas
- renders the side-by-side reconciliation tables
